# Skill Marketplace

Internal marketplace for company-approved Claude plugins. Each plugin is a self-contained folder with a manifest and a single skill. The marketplace is auto-generated by CI on every deploy.

---

## How it works

```
public/
  plugins/
    sql-writer/
      plugin.json     ← plugin metadata (owned by the plugin)
      SKILL.md        ← the skill Claude loads
  tag-taxonomy.json   ← controlled tag vocabulary
  marketplace.json    ← AUTO-GENERATED by CI (never edit manually)
```

`marketplace.json` is generated by `scripts/generate-marketplace.js` on every deploy. It contains the full list of approved plugin IDs and a tag index for fast discovery. Contributors never touch it.

---

## Run locally

```bash
npm install
npm run dev        # generates marketplace.json then starts dev server
```

Open `http://localhost:5173`.

To run the generator alone:
```bash
npm run generate
```

## Add a plugin

Create a folder under `public/plugins/[your-plugin-id]/` with two files:

**plugin.json**
```json
{
  "id": "your-plugin-id",
  "name": "Human Readable Name",
  "version": "1.0",
  "description": "What this skill enables Claude to do.",
  "category": "Data",
  "tags": ["tag1", "tag2"],
  "author": "Your Team",
  "approved": true,
  "added": "2026-01-15",
  "example_prompt": "An example of what a user would ask",
  "skill": { "path": "SKILL.md" }
}
```

Valid categories: `Data`, `Productivity`, `Communication`, `Strategy`
Tags: lowercase, hyphens-for-spaces, max 5, must be in `tag-taxonomy.json`

**SKILL.md**
```markdown
---
id: your-plugin-id
version: 1.0
author: Your Team
---

# Skill Name

Instructions for Claude go here.
```

Open a pull request. CI validates, generates `marketplace.json`, and posts a preview URL. Admin merges → production deploys in ~30 seconds.

---

## Validation rules

The generator enforces three layers before writing `marketplace.json`:

**Layer 1 — Schema** (plugin.json)
All required fields present, valid category, boolean approved, tags are array.

**Layer 2 — Structure** (per plugin folder)
plugin.json id matches folder name, SKILL.md exists at skill.path, SKILL.md has valid frontmatter.

**Layer 3 — Tags**
All tags lowercase, no spaces (use hyphens), max 5, all present in tag-taxonomy.json, no duplicates within plugin.

Plugins with `approved: false` are skipped — they don't appear in marketplace.json.

---

## Adding a new tag

Edit `public/tag-taxonomy.json` and add your tag to the array. Include this change in your plugin PR. Admin reviews it deliberately — tags are shared vocabulary across all plugins.

## CI/CD pipeline

| Event | What happens |
|---|---|
| Pull request | Generate → Validate → Build → Preview URL on PR |
| Merge to main | Generate → Validate → Build → Production deploy |
| Validation fails | Build blocked, PR cannot merge |

## Cloudflare setup (one time)

1. Push repo to GitHub
2. Go to pages.cloudflare.com → Create project → Connect repo
3. Build command: `npm run build` | Output directory: `dist`
4. Add GitHub secrets: `CLOUDFLARE_API_TOKEN`, `CLOUDFLARE_ACCOUNT_ID`

---

## Enable in-session discovery

1. Open `MARKETPLACE_SKILL.md`
2. Replace `MARKETPLACE_URL` with your Cloudflare URL
3. Save it as a skill in Claude (Customize → Skills → +)

Users can then ask Claude: *"Is there a plugin for writing SQL?"* and Claude will check the marketplace, confirm the match, and load the skill immediately.
